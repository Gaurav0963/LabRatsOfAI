2. Alpha-Beta Pruning

Using the same game tree, apply Alpha-Beta Pruning to find the optimal value. Assume children are evaluated left-to-right. Which nodes, if any, are pruned?

Expected Input

Game Tree:
- Root (MAX): α=-∞, β=+∞
  - Node A (MIN)
    - Terminal Node: 3
    - Terminal Node: 5
  - Node B (MIN)
    - Terminal Node: 2
    - Terminal Node: 9

Expected Output

The optimal value is 3. The node with value 9 is pruned.

Explanation:
1. Root (MAX) starts with α=-∞, β=+∞.
2. Go to Node A (MIN). It gets value min(3, 5) = 3.
3. Return 3 to Root. Root (MAX) updates its alpha: α = 3.
4. Go to Node B (MIN), passing down α=3, β=+∞.
5. Node B evaluates its first child (2). 
6. At Node B, the current value (2) is less than α (3).
7. PRUNING: Since MIN (at B) can already achieve a value of 2 (which is worse for MAX than the 3 it already has from A), MAX will never choose the path to B. There's no need to check B's other children.
8. The node with value 9 is pruned.
9. Final value at Root is max(3, ...) which remains 3.
